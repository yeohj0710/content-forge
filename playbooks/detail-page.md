# Playbook — 제품 상세페이지

> 제품 상세페이지(스마트스토어·자사몰·쿠팡 등) 작업 시 이 문서를 blueprint 작성 전에 읽는다.
> `standards/design-standards.md`의 공통 기준 위에 얹히는 전용 기준이다.

---

## 0. 이 산출물의 본질

상세페이지는 **세일즈 문서**다. 예쁜 것은 수단이고, 목적은 스크롤을 붙잡고 구매를 설득하는 것.
웹 랜딩페이지의 미학(넓은 여백, 작고 세련된 텍스트)을 가져오면 반드시 실패한다.
커머스 상세페이지의 문법은 **큰 타이포·높은 밀도·숫자·증거·반복 CTA**다.

작업 전 자문: "이 페이지에서 고객이 사야 할 이유를 5초 안에 숫자로 말할 수 있는가?"
research.md의 셀링포인트 없이는 이 문서를 시작할 수 없다(AGENTS.md Phase 1 필수).

---

## 1. 캔버스 규격

- **작업 폭 1000px 고정.** (스마트스토어 권장 860px — 1000px로 작업 후 축소 export하면 선명도 확보)
- 섹션 프레임: 폭 1000px, 세로 1000~1600px. 세로 오토레이아웃으로 스택.
- 분할 export 대비: 섹션 경계 = 분할 경계. 플랫폼 이미지 1장당 세로 상한(스마트스토어 기준
  긴 이미지는 분할 업로드) 고려. 텍스트·비주얼이 경계에 걸리지 않게.
- 전체 길이 가이드: 12~16섹션, 세로 합계 14,000~22,000px 수준(레퍼런스급 페이지의 밀도).

## 2. 타이포 스케일 ★ (폭 1000px 기준 — 절대 어기지 말 것)

| 역할 | 크기 | 비고 |
|---|---|---|
| 어깨 라벨(eyebrow, "POINT 01") | 24~28px | 자간 +5~10%, 대문자/컬러 |
| 섹션 헤드라인(H1) | **60~90px** | 훅 섹션은 최대 100px, Bold 이상 |
| 서브 헤드라인(H2) | 40~48px | |
| 본문 | **26~32px** | 행간 150~160% |
| 캡션·주석·디스클레이머 | 20~24px | **20px 미만 절대 금지** |
| 숫자 셀링포인트(함량·배수·%) | **90~160px** | 섹션 내 최대 크기. 단위는 절반 크기로 |
| 비교표 셀 텍스트 | 26~30px | 표 헤더는 28~34px Bold |

검산 규칙: 완성 후 아무 섹션이나 골라 본문 px ÷ 1000 ≥ 0.026인지 확인.
과거 실패 사례: 본문 16~18px로 조립 → 모바일에서 판독 불가 → 전면 리워크.

## 3. 설득 아키텍처 — 표준 섹션 시퀀스

research.md의 셀링포인트를 아래 뼈대에 배치한다. 순서·개수는 제품에 맞게 조정하되,
**각 섹션 = 셀링포인트 1개 + 그 근거**라는 원칙은 불변이다.

| # | 섹션 | 역할 | 필수 요소 |
|---|---|---|---|
| 1 | 훅 | 3초 안에 "이게 뭐고 왜 대단한지" | 제품 실사 연출컷 + 한 줄 컨셉 + 최강 숫자 1개 |
| 2 | 핵심 요약 | 셀링포인트 3~4개 아이콘 요약 | 아이콘 행(탄력/보습 등), 큰 숫자 |
| 3 | 문제 공감 | 타겟의 페인포인트 | 공감 카피(감성), 실사/인물컷 |
| 4 | 솔루션 선언 | 제품 = 해답 | 제품컷 + 카테고리 재정의 카피 |
| 5~7 | Point 01~03 | 차별점 각개격파(함량·원료·설계자 등) | **넘버링 뱃지 + 숫자 + 증거 이미지** |
| 8 | 비교 | 경쟁사/기존 방식 대비 | 비교표 또는 나란히 비교(경쟁품은 블러/실루엣) |
| 9 | 과학적 근거 | 논문·시험 데이터 | 논문 캡처/그래프 + 출처 표기 + 디스클레이머 |
| 10 | 신뢰 | 인증·제조·이력 | **인증서 실물 스캔 그리드**, 공장/약국 사진 |
| 11 | 사회적 증거 | 리뷰·판매량·입점처 | 리뷰 캡처, 별점, 입점 로고 |
| 12 | 섭취/사용법 | 사용 장벽 제거 | 스텝 그래픽(1-2-3), 실사 |
| 13 | 구성·FAQ | 구매 직전 의문 해소 | 구성품 사진+가격 구성, Q&A 3~5개 |
| 14 | 마무리 CTA | 구매 밀어붙이기 | 감성 카피 + 제품컷 + CTA 버튼 그래픽 + 법적 문구 |

## 4. 셀링포인트 표현 규칙

- **숫자가 왕이다.** "고함량" ✕ → "300,000ppm" ✓. "많은 분들이 선택" ✕ → "리뷰 1,200개" ✓.
  숫자는 §2의 90~160px 스케일로, 섹션에서 가장 먼저 보이게.
- **근거 없는 주장 금지.** 모든 강조 카피에는 같은 섹션 안에 근거(수치 출처·인증·논문·실측 사진)가
  붙어야 한다. 근거 에셋이 없으면 → 매니페스트로 요구하거나 해당 카피를 버린다. 지어내지 않는다.
- **비교가 설득한다.** 경쟁사 평균·기존 방식(시술 등)과의 비교 섹션을 최소 1개 넣는다.
  경쟁품 실물은 블러/흑백 처리로 명예훼손 리스크를 피한다.
- 감성 카피(손글씨체 등)는 페이지당 1~2곳의 포인트로만.

## 5. 배경 전략 ★ (GPT Image 비율 한계 대응)

**기본값은 Figma 네이티브다. 생성 이미지는 예외다.**

1. **섹션 배경 기본**: Figma 단색/그라데이션(브랜드 컬러군 명도 그라데이션) + 필요 시
   시임리스 텍스처 1장 반복. 페이지 전체 톤 통일은 배경이 아니라 디자인 토큰이 만든다.
2. **실사 풀블리드 섹션**(모델 얼굴, 원료 매크로, 제조 현장 등 "사진이 곧 메시지"인 섹션):
   - **섹션을 2:3 이하(1000×1500 이하)로 설계**해서 GPT Image 1024×1536 1장과 1:1 매칭시킨다.
   - 또는 사진을 섹션 상단 2:3 영역에만 깔고, 하단은 단색으로 페이드시켜 다음 섹션과 잇는다.
   - 프롬프트에 페이드 지시 필수: "하단 20%는 #hex로 부드럽게 페이드아웃".
3. **절대 금지**: 2:3보다 긴 프레임을 생성 이미지 1장으로 채우기(스트레치·업스케일 흔적),
   아웃페인팅 이어붙이기(톤 드리프트).
4. **최고의 배경은 실제 제품 사진이다.** 스튜디오 연출컷(제품+원물+액체 연출)을 P 에셋으로
   요구하는 것을 기본값으로 한다. 생성 이미지는 실사가 없을 때의 차선책이다.

## 6. 필수 P 에셋 기본 목록 (매니페스트에 기본 포함할 것)

- 제품 연출컷(박스+스틱+내용물 연출, 최소 3컷) / 정면·사선 누끼
- **인증서 실물 스캔 전부** (FSSC, GMP, HACCP, 특허 등 — 텍스트 뱃지로 때우지 않는다)
- 논문·시험성적서 캡처(출처 명기 가능한 것)
- 리뷰 캡처·판매량·입점처 증빙
- Before/After 또는 질감 시연 컷(광고 심의 확인 후)
- 브랜드 로고 원본, 지정 폰트

## 7. 표현 심의 주의 (식품·화장품·건기식)

- 의약품 오인 표현 금지: "치료", "재생", "병이 낫는다" 류 → "개선에 도움", "~에 도움을 줄 수 있음".
- 임상·논문 인용 시: 출처 표기 + "개인차가 있을 수 있습니다" 디스클레이머(캡션 스케일).
- Before/After는 "연출된 이미지" 문구 필요 여부 확인.
- 최종 문구 책임은 사용자에게 있음을 공유 시 고지하고, 위험 표현은 대안과 함께 표시한다.

## 8. 셀프 QA 추가 항목 (공통 체크리스트에 더해)

- [ ] 본문 ≥ 26px, 캡션 ≥ 20px (폭 1000px 기준) — 전 섹션 검산
- [ ] 축소 판독 테스트(maxDimension 400) 통과 — 숫자 셀링포인트가 축소에서도 튀는가
- [ ] 섹션마다 근거 요소(숫자·이미지 증거) 존재하는가
- [ ] 카드·표 바닥에 빈 공간 없는가 (hug 확인)
- [ ] 레퍼런스와 나란히 비교 — 밀도·타이포 크기·증거의 양에서 밀리지 않는가
- [ ] 생성 배경 스트레치 흔적 없는가, 섹션 이음새 자연스러운가
- [ ] 심의 위험 표현 목록 작성했는가
