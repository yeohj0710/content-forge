# Playbook — 단장 광고 이미지 (SNS 광고 · 배너 · 썸네일)

> 1~N장짜리 독립 이미지(피드 광고, 배너, 유튜브 썸네일, 스토어 대표 이미지) 작업 시
> blueprint 작성 전에 읽는다. `standards/design-standards.md` 위에 얹히는 전용 기준이다.

---

## 0. 이 산출물의 본질

광고 이미지는 **3초 싸움**이다. 피드를 스크롤하는 엄지를 멈추게 하는 것이 유일한 목적.
상세페이지처럼 설득을 쌓을 지면이 없다 — **메시지 1개, 비주얼 1개, 3초.**

## 1. 캔버스 규격 (요청 플랫폼에 맞춰 확정)

| 용도 | 비율 | 작업 크기 |
|---|---|---|
| 피드 광고(인스타·페북) | 1:1 / 4:5 | 1080×1080 / 1080×1350 |
| 스토리·릴스 커버 | 9:16 | 1080×1920 |
| 유튜브 썸네일 | 16:9 | 1280×720 |
| 가로 배너 | 플랫폼별 | 요청 스펙 |

- 9:16은 상하 세이프존(상단 250px, 하단 300px — UI 겹침 구간)에 핵심 요소 금지.
- 시리즈물(광고 N종)은 같은 토큰·같은 레이아웃 시스템으로 연작감을 만든다.

## 2. 타이포 스케일 (1080px 폭 기준)

| 역할 | 크기 | 비고 |
|---|---|---|
| 메인 카피(H1) | **90~150px** (폭의 8~14%) | 2줄 이내. 썸네일은 상한까지 |
| 서브 카피 | 40~60px | 1줄. 없어도 된다 |
| CTA·뱃지 | 32~44px | 필요 시만 |

- **위계는 2단까지만.** 메인+서브. 셋째 위계가 필요하면 카피가 너무 많은 것이다.
- 텍스트 총량: 15단어 이내. 플랫폼 텍스트 비중 정책(메타 20% 권고) 감안.
- 축소 판독 테스트를 **200px 폭**으로 한다(피드 썸네일 크기). 이 크기에서 메인 카피가
  읽히고 제품이 보여야 통과.

## 3. 레이아웃 패턴 (기본 3종)

1. **제품 히어로**: 제품 연출컷 중앙 대형 + 상단/하단 카피 밴드.
2. **비포애프터/비교**: 좌우 분할 + 중앙 결과 카피.
3. **텍스트 훅**: 대형 카피가 주인공, 제품은 하단 소형(호기심 유발형).

시리즈 광고는 패턴 1~2종을 반복 변주한다(이미지마다 새 레이아웃 금지 — 연작감 우선).

## 4. 배경·비주얼 전략

- 1:1, 4:5는 GPT Image 비율 상한 안이므로 **풀블리드 생성 배경이 기본값**이다(생성은 사실상
  무제한 — AGENTS.md §2-4). 단색·그라데이션만의 배경은 의도적 미니멀 선택일 때만. 단 카피 영역
  네거티브 스페이스 지시 필수(프롬프트에 "상단 1/3 단순하게" 등).
- 카드뉴스처럼 본문이 흰 정보 박스 위에 얹히는 지면도 풀블리드 배경 적극 활용 — 박스가
  가독성을 지켜주므로, 배경은 표 위에서도 안전하다. 표·리스트가 덮는 장은 피사체를 모서리에
  작게 두는 "조용한 배경"으로 변주해 연작감을 유지한다.
- 9:16은 생성 상한(2:3)을 넘는다 → 생성 이미지는 중앙 2:3 영역에, 상하는 Figma
  그라데이션으로 연장(페이드 지시 필수). 또는 배경 전체를 Figma 네이티브로.
- 제품 누끼는 배경과 대비되는 크기로 과감하게(폭의 50~80%). 소품처럼 작게 두지 않는다.
- 그림자·반사 등 그라운딩 처리로 "붙여넣은 느낌"을 제거한다.

## 5. 셀프 QA 추가 항목

- [ ] 9:16 세이프존 침범 없는가
- [ ] 시리즈물: 나란히 놓았을 때 연작으로 보이는가
- [ ] 플랫폼 텍스트 비중·심의 기준(과장 표현) 위반 없는가
